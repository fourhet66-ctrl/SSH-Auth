# ssh-auth v0.1.0

Draft release notes for the first public release of `ssh-auth`.

## Highlights

- Manage SSH profiles for Codex remote development.
- Stable `codex-active` alias: keep Codex pointed at one SSH host while switching the real remote host behind it.
- Core commands: `list`, `add`, `switch`, `check`, and `connect`.
- No password storage: `ssh-auth` does not store SSH passwords, private keys, or Codex private database data.

## Core workflow

```shell
ssh-auth add gpu01 --login "ssh ubuntu@1.2.3.4 -p 22" --setup-key
ssh-auth list
ssh-auth switch 01
ssh-auth check
ssh-auth connect 01
```

After enabling Codex remote support, Codex can use:

```text
codex-active
```

Switching profiles rewrites the managed OpenSSH config for `codex-active`, so Codex does not need to change its configured host when you move between GPU servers or devboxes.

## Security notes

- No password storage.
- Password prompts, when needed for key setup, are delegated to OpenSSH tools such as `ssh-copy-id`.
- Missing SSH keys are not generated unless requested with `--generate-key`.
- Profile metadata may include hostnames, usernames, ports, key paths, timestamps, and resource summaries; do not publish real local registry files.
